<task type="auto" tdd="true">
  <name>Task 2: Git change detection module with TDD</name>
  <files>src/scanner/change-detection.ts, tests/change-detection.test.ts, src/scanner/index.ts</files>
  <behavior>
    - getCurrentSha returns trimmed HEAD SHA string for a valid git repo
    - getChangedFiles returns empty array when stored SHA equals current SHA
    - getChangedFiles returns list of changed file paths when SHAs differ
    - needsScan returns true when no stored SHA exists (first run)
    - needsScan returns false when stored SHA equals current HEAD
    - needsScan returns true when stored SHA differs from current HEAD
    - getChangedFiles throws/handles error gracefully when repoPath is not a git repo
  </behavior>
  <action>
Create src/scanner/change-detection.ts:

- Import `exec` from `node:child_process` and `promisify` from `node:util`
- `const execAsync = promisify(exec)`
- `getCurrentSha(repoPath: string): Promise<string>` — runs `git rev-parse HEAD` with `{ cwd: repoPath }`, returns trimmed stdout
- `getChangedFiles(repoPath: string, storedSha: string): Promise<string[]>` — if storedSha is empty/undefined, return empty array (caller should do full scan). Otherwise run `git diff --name-only ${storedSha} HEAD` with cwd, split on newline, filter empty strings
- `needsScan(repoPath: string, storedSha: string | undefined): Promise<{ needed: boolean; currentSha: string; changedFiles: string[] }>` — if no storedSha, return `{ needed: true, currentSha, changedFiles: [] }` (full scan). If SHA same, return `{ needed: false, ... }`. If different, return changed files.

For tests: use the actual sawyer-support-docs repo as the test git repo (it IS a git repo). For the "different SHA" test, use an older SHA from the git log. For "not a git repo" test, use a temp directory.

Update src/scanner/index.ts to also export from change-detection.ts.
  </action>

  </verify>
