<task type="auto">
  <name>Task 1: Wire generate command with feature map loading, filtering, and per-feature generation loop</name>
  <files>src/commands/generate.ts</files>
  <action>
    Replace the stub in `src/commands/generate.ts` with a full implementation:

    **Imports:**
    - `loadConfig` from `../config/index.js`
    - `readFeatureMap` from `../scanner/index.js`
    - `runGeneration`, `writeArticle` from `../generator/index.js`
    - `createLogger` from `../ui/logger.js`

    **Command options (update existing):**
    - Keep `--dry-run` option (preview mode)
    - Add `--features <slugs>` option with description "Comma-separated feature slugs to generate"
    - Remove `--mobile`, `--dashboard`, `--platform` options (those belong to scan command, not generate)

    **Action handler implementation:**
    1. `const config = loadConfig()` — load and validate config
    2. `const logger = createLogger(false, false)` — create logger (no verbose/quiet flags on generate for now)
    3. `const cwd = process.cwd()`
    4. `const featureMap = readFeatureMap(cwd)` — read feature map from .sawyer-docs/
    5. If `!featureMap`: `logger.error('No feature map found. Run "sawyer-docs scan" first.')` and `process.exit(1)`
    6. `let features = featureMap.features`
    7. If `options.features`: split by comma, trim each, filter features array to only those whose `slug` is in the set. If filtered result is empty, `logger.warn('No matching features found for the provided slugs.')` and `process.exit(0)`
    8. `logger.info(\`Generating articles for ${features.length} feature(s)...\`)`
    9. If `options.dryRun`: list each feature (name, slug, area) with `logger.info()`, then `logger.success('Dry run complete — no articles generated.')` and return
    10. Track `let generated = 0; let failed = 0;`
    11. Loop: `for (let i = 0; i < features.length; i++)`
        - `const feature = features[i]!`
        - `const spinner = logger.spinner(\`Generating: ${feature.name} (${i + 1}/${features.length})...\`)`
        - try: `const content = await runGeneration(feature, config.model)`
        - `const relPath = writeArticle(cwd, feature, content)`
        - `spinner.succeed(\`Generated: ${relPath}\`)`
        - `generated++`
        - catch (err): `spinner.fail(\`Failed: ${feature.name} — ${(err as Error).message}\`)` + `failed++`
    12. After loop: `logger.success(\`Generation complete: ${generated} generated, ${failed} failed.\`)`
    13. If `failed > 0`: `process.exit(1)` (signal partial failure to caller)

    **Important details:**
    - The entire action handler body is wrapped in try/catch for config loading errors (existing pattern)
    - Sequential feature processing (not parallel) per RESEARCH.md anti-pattern guidance
    - `--features` takes slugs (not names) per RESEARCH.md Pitfall 4
    - Do NOT use Commander's `.option('--verbose')` or `.option('--quiet')` — keep it simple for Phase 3
  </action>
