  <name>Task 1: File path contract — slug utility and path builder with tests</name>
  <files>src/paths/slugify.ts, src/paths/paths.ts, src/paths/index.ts, tests/paths.test.ts</files>
  <action>
1. Create src/paths/slugify.ts:
   - Import slugify from '@sindresorhus/slugify'
   - Export `buildSlug(input: string): string` that calls `slugify(input, { separator: '-', lowercase: true })`
   - This is the stable public contract — same input always produces same output
   - German umlauts are handled by default: ae for ae, oe for oe, ue for ue

2. Create src/paths/paths.ts:
   - Import `join` from 'node:path'
   - Import `SupportedLang` from '../config/schema.js' (use the type from config, not a local copy)
   - Export `buildArticlePath(lang: SupportedLang, featureArea: string, slug: string): string`
   - Returns `join('docs', lang, featureArea, `${slug}.md`)`
   - The function does NOT call buildSlug — callers are responsible for passing a pre-slugified value. This keeps the path builder pure and predictable.

3. Create src/paths/index.ts — barrel re-export:
   - Export `buildSlug` from './slugify.js'
   - Export `buildArticlePath` from './paths.js'
   - Re-export `SupportedLang` from '../config/schema.js' for convenience
   - IMPORTANT: Use `.js` extensions in all imports

4. Write tests/paths.test.ts:
   - Test: buildSlug('Benutzer-Authentifizierung') returns 'benutzer-authentifizierung'
   - Test: buildSlug('Passwort zurucksetzen') handles spaces (becomes 'passwort-zurucksetzen')
   - Test: buildSlug('Club Ubersicht') produces 'club-ubersicht'
   - Test: buildSlug with German umlauts: 'Ubersicht' -> contains no u-umlaut characters
   - Test: buildSlug is idempotent — buildSlug(buildSlug(input)) === buildSlug(input)
   - Test: buildSlug produces only URL-safe characters (matches /^[a-z0-9-]+$/)
   - Test: buildArticlePath('de', 'authentication', 'login') returns 'docs/de/authentication/login.md'
   - Test: buildArticlePath produces consistent structure across languages — same featureArea and slug for 'de' and 'nl' differ only in language segment
   - Test: buildArticlePath('en', 'payments', 'checkout') returns 'docs/en/payments/checkout.md'
