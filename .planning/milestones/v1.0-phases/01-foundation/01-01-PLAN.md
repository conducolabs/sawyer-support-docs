<task type="auto" tdd="true">
  <name>Task 2: Config schema, loader with .env overlay, and CLI entry point</name>
  <files>src/config/schema.ts, src/config/loader.ts, src/config/index.ts, src/ui/logger.ts, src/bin/cli.ts, src/commands/generate.ts, src/commands/translate.ts, src/commands/scan.ts, tests/config.test.ts</files>
  <action>
1. Create src/config/schema.ts:
   - Define `SUPPORTED_LANGS = ['de', 'en', 'nl', 'tr', 'uk'] as const` and export it
   - Define `SupportedLang` type from the const array
   - Define `ConfigSchema` using Zod v4:
     - `languages`: `z.array(z.enum(SUPPORTED_LANGS)).default(['de', 'en', 'nl', 'tr', 'uk'])`
     - `model`: `z.string().default('claude-sonnet-4-5')`
     - `repos`: `z.object({ mobile: z.string(), dashboard: z.string(), platform: z.string() })`
     - `deepl_api_key`: `z.string().min(1, 'DEEPL_API_KEY is required')`
     - `anthropic_api_key`: `z.string().min(1, 'ANTHROPIC_API_KEY is required')`
   - Export `type Config = z.infer<typeof ConfigSchema>`

2. Create src/config/loader.ts:
   - Import dotenv, readFileSync, resolve, ConfigSchema
   - `loadConfig(cwd = process.cwd())`:
     - Call `dotenv.config({ path: resolve(cwd, '.env') })` FIRST
     - Read and parse `sawyer-docs.config.json` from cwd
     - Overlay env vars: `process.env.DEEPL_API_KEY` and `process.env.ANTHROPIC_API_KEY` win over JSON values
     - Also overlay `process.env.SAWYER_DOCS_MODEL` onto model field if present
     - Also overlay `process.env.SAWYER_DOCS_LANGUAGES` (comma-separated) onto languages if present
     - Use `ConfigSchema.safeParse(merged)` — NOT `.parse()`
     - On failure: format issues as `"  - {path}: {message}"` per line, throw Error with `"Configuration invalid:\n{issues}"`
     - On success: return `result.data` (fully typed Config)
   - Handle missing config file: throw clear error "sawyer-docs.config.json not found in {cwd}"
   - Handle invalid JSON: throw clear error "sawyer-docs.config.json contains invalid JSON"

3. Create src/config/index.ts — barrel re-export:
   - Export Config, ConfigSchema, SUPPORTED_LANGS, SupportedLang from schema.js
   - Export loadConfig from loader.js
   - IMPORTANT: Use `.js` extensions in all imports (ESM requirement)

4. Create src/ui/logger.ts:
   - Import ora and picocolors
   - Export a `createLogger(verbose: boolean, quiet: boolean)` function returning an object with:
     - `spinner(text: string)`: creates and starts an ora spinner, returns it
     - `info(msg: string)`: prints with blue prefix (only if not quiet)
     - `warn(msg: string)`: prints with yellow prefix
     - `error(msg: string)`: prints with red prefix
     - `success(msg: string)`: prints with green prefix
   - Default verbosity: normal (spinner visible, summary shown). --verbose adds detail. --quiet suppresses non-errors.

5. Create src/bin/cli.ts:
   - Add hashbang: `#!/usr/bin/env node`
   - Create Commander program named 'sawyer-docs'
   - Add global options: `-v, --verbose`, `-q, --quiet`, `--config <path>` (default: 'sawyer-docs.config.json')
   - Register three subcommands via `.addCommand()`
   - Call `program.parse()`

6. Create src/commands/generate.ts — stub:
   - Export `generateCommand = new Command('generate')` with description "Generate support articles from codebase scan"
   - Add options: `--mobile <path>`, `--dashboard <path>`, `--platform <path>`, `--dry-run`

7. Create src/commands/translate.ts — stub:
